# دليل بنية وتصميم نظام إدارة إنجاز (Enjaz Admin Panel Architecture)

مرحباً بك! هذا الدليل يوضح بالتفصيل كيف تم تصميم وبرمجة نظام إدارة إنجاز للحلول الذكية (**Enjaz-Admin**). سيفيدك هذا التحليل بشكل كبير لتصميم أنظمة مماثلة بنفس الكفاءة والأمان والسرعة.

---

## 1. لغة البرمجة والتقنيات الأساسية (Tech Stack)

النظام مبني على فلسفة **الخفة والسرعة والاعتمادية** بدون تعقيد إطارات العمل الضخمة (Frameworks):

*   **لغة البرمجة الأساسية**: **PHP (Vanilla PHP)** بنمط برمجي منظم ونظيف يسهل فهمه وتعديله.
*   **قاعدة البيانات**: **MySQL / MariaDB** مع استخدام مكتبة **PDO (PHP Data Objects)** لإدارة الاتصال والاستعلامات بشكل آمن تماماً.
*   **الواجهة الأمامية (Frontend)**:
    *   **HTML5 & CSS3**: تصميم مخصص بالكامل (Custom CSS) يدعم الاتجاه من اليمين لليسار (**RTL**) وبخط **Cairo** المخصص للغة العربية.
    *   **JavaScript (Vanilla JS)**: للتفاعلات الديناميكية (مثل إظهار وإخفاء كلمة المرور، وعدادات التحميل أثناء إرسال النماذج) وطلبات **AJAX** للخلفية.
    *   **FontAwesome 6**: لتوفير أيقونات واجهة المستخدم.

## 2. بنية المجلدات وهيكل النظام (Directory Structure)

يتبع النظام هيكلية واضحة ومقسّمة حسب الوظائف (Modular Structure):

```text
enjaz-admin/
│
├── config/               # ملفات الإعدادات والاتصال بقاعدة البيانات
│   ├── .htaccess         # حماية الملفات داخل هذا المجلد من الوصول الخارجي
│   ├── app.php           # إعدادات التطبيق العامة، الصلاحيات، والتحقق التلقائي من تحديثات قاعدة البيانات
│   └── db.php            # إعداد اتصال PDO الآمن وإدارته
│
├── database/             # ملفات قواعد البيانات والجداول وخطط الهجرة (Migrations)
│   ├── .htaccess         # حماية قواعد البيانات من التحميل المباشر
│   └── enjaz_admin.sql   # الهيكل الأساسي للجداول وبيانات الإعدادات
│
├── includes/             # الدوال العامة ونظام المصادقة المشترك
│   ├── .htaccess         # منع الوصول المباشر لملفات البرمجة الداخلية
│   ├── auth.php          # إدارة الجلسات، التحقق من الصلاحيات، تسجيل الدخول والخروج والتأمين ضد الاختراق
│   ├── functions.php     # الدوال المساعدة (تنسيق المبالغ، التواريخ، توليد شارات الحالة Badge، حماية XSS)
│   ├── header.php        # القالب العلوي للوحة التحكم وشريط التنقل
│   └── footer.php        # القالب السفلي للوحة التحكم وملفات الـ Javascript المشتركة
│
├── api/                  # نقاط النهاية (Endpoints) لعمليات AJAX السريعة
│   ├── check-username.php
│   ├── service-plans.php
│   └── whatsapp.php
│
├── whatsapp/             # نظام إدارة وإرسال رسائل الواتساب
│   ├── cron.php          # سكريبت الخلفية (Cron Job) المسؤول عن إرسال الرسائل المجدولة تلقائياً
│   ├── bulk.php          # واجهة لإرسال رسائل جماعية للعملاء
│   └── schedules.php     # إدارة الرسائل المجدولة وقائمة الانتظار
│
├── [modules]/            # مجلدات الوحدات المختلفة (كل وحدة تحتوي على ملفات CRUD الخاصة بها)
│   ├── clients/          # إدارة العملاء (إضافة، تعديل، حذف، عرض)
│   ├── services/         # إدارة الخدمات المتاحة
│   ├── subscriptions/    # إدارة الاشتراكات
│   ├── payments/         # تسجيل وإدارة المدفوعات وتخزين الإيصالات
│   └── reports/          # تقارير التجديدات والتقارير المالية والتحليلات
│
├── index.php             # نقطة الدخول (توجيه المستخدم للوحة التحكم أو صفحة تسجيل الدخول)
├── login.php             # واجهة تسجيل الدخول المؤمنة بالكامل
└── logout.php            # تسجيل الخروج الآمن وتدمير الجلسة
```

## 3. التدابير الأمنية (Security & Hardening)

تعتبر الحماية في هذا النظام من أقوى النقاط التي تم التركيز عليها، وهي مقسمة كالتالي:

### أ. الحماية ضد ثغرة حقن استعلامات قاعدة البيانات (SQL Injection)
*   يتم استخدام **PDO Prepared Statements** بشكل إلزامي في جميع الاستعلامات.
*   إعداد `PDO::ATTR_EMULATE_PREPARES => false` للتأكد من أن الاستعلامات يتم تجهيزها وتنفيذها حقيقياً على خادم MySQL مما يمنع تمرير أي أوامر خبيثة نهائياً.

### ب. الحماية ضد ثغرات XSS (حقن الأكواد البرمجية بالواجهة)
*   تم إنشاء دالة مساعدة باسم `e($value)` والتي تقوم بتمرير أي بيانات نصية قادمة من قاعدة البيانات أو المستخدم عبر دالة `htmlspecialchars` بتشفير `UTF-8` لمنع تشغيل أي أكواد HTML أو Javascript خبيثة في المتصفح.

### ج. حماية الجلسات والمصادقة (Session Security & Authentication)
في ملف `config/app.php` يتم تهيئة الجلسة بالخصائص الأمنية التالية:
*   `session.cookie_httponly`: تفعيلها يمنع لغة Javascript (وثغرات XSS إن وجدت) من قراءة ملف تعريف الجلسة (Session Cookie).
*   `session.cookie_samesite = Strict`: تمنع تماماً هجمات **CSRF** (تزوير الطلبات عبر المواقع الخارجية).
*   `session.use_strict_mode`: تمنع المهاجمين من استخدام معرفات جلسة مخترقة أو غير مهيأة من طرف الخادم.
*   `session_regenerate_id(true)`: يتم استدعاؤها فور تسجيل الدخول الناجح لمنع هجمات **Session Fixation** (تثبيت الجلسة).
*   **مهلة عدم النشاط (Inactivity Timeout)**: النظام يفحص وقت آخر نشاط للمستخدم وإذا تجاوز 15 دقيقة (900 ثانية)، يتم تسجيل خروجه تلقائياً لحماية حسابه إذا ترك الجهاز مفتوحاً.

### د. حماية النماذج (CSRF Protection)
*   توليد توكن أمان مشفر عشوائي (`csrf_token`) مخزن في الجلسة لكل مستخدم.
*   يتم زرع حقل مخفي في كل نموذج POST باستخدام الدالة `csrfField()`.
*   يتم التحقق من تطابق التوكن مع كل طلب مرسل باستخدام دالة `verifyCsrf()` عبر مقارنة آمنة زمنياً (`hash_equals`) لمنع هجمات التخمين.

### هـ. خصوصية الأخطاء (Error Privacy)
*   تم تغليف استدعاءات PDO بكتلة `try-catch`. عند حدوث خطأ في الاتصال، يتم تخزين تفاصيل الخطأ التقني داخلياً في ملف سجل الأخطاء بالسيرفر (`error_log`)، بينما يتم عرض رسالة عامة ومحترمة للمستخدم بدون كشف كلمات المرور أو بنية الجداول.

### و. حماية الملفات عبر السيرفر (.htaccess)
*   توجد ملفات `.htaccess` داخل مجلدات الإعدادات والبرمجة الحساسة مثل (`config`, `includes`, `database`) تمنع تماماً الوصول المباشر لأي ملف بداخلها عبر الويب (HTTP 403 Forbidden).

## 4. تصميم قاعدة البيانات وآلية الإصلاح الذاتي (Auto-migrations)

يتميز النظام بميزة ذكية جداً في ملف `config/app.php`:
*   عند إقلاع التطبيق، يقوم بطلب فحص سريع لبنية الجداول في قاعدة البيانات.
*   إذا وجد حقولاً جديدة تمت إضافتها في التحديثات (مثل حقل الدومين أو حقول الواتساب والمصروفات)، يقوم النظام بتشغيل أوامر `ALTER TABLE` أو `CREATE TABLE` تلقائياً.
*   هذه الآلية توفر تجربة ممتازة عند رفع النظام على سيرفر جديد أو تحديثه دون الحاجة لتشغيل سكريبتات SQL يدوية في كل مرة.

## 5. تجربة المستخدم والتصميم البصري (UI/UX)

تم تصميم النظام ليكون مريحاً وعملياً ومبهراً بصرياً:

*   **جماليات زجاجية (Glassmorphism)**: استخدام تباين داكن أنيق جداً في شاشة تسجيل الدخول وخلفيات متدرجة مع خاصية `backdrop-filter: blur(20px)` لتعطي طابعاً حديثاً وفخماً للغاية.
*   **التغذية البصرية والتفاعل الفوري (Micro-animations)**:
    *   اهتزاز نافذة تسجيل الدخول (Shake) عند إدخال بيانات خاطئة لتنبيه المستخدم.
    *   تحويل زر الإرسال إلى وضعية الانتظار وجعل الأيقونات تدور (Spinner) لمنع تكرار الضغط المزدوج أثناء معالجة الطلبات.
    *   إمكانية إظهار وإخفاء كلمات المرور بلمسة واحدة.
*   **التنبيهات السريعة (Flash Messages)**: نظام لعرض رسائل النجاح أو الأخطاء في أعلى الصفحات مع أيقونات ملونة مناسبة لنوع الرسالة وتختفي بمجرد الانتقال.
*   **شارات الحالة الذكية (Badges)**: تحويل حالات الاشتراكات والفواتير إلى ألوان بصرية مريحة للعين (الأخضر للنشط والمسدد، البرتقالي للمقترب من الانتهاء، الأحمر للمنتهي).

## 6. نظام إرسال الواتساب وجدولة الرسائل (WhatsApp Queue System)

يحتوي النظام على بوابة مخصصة للربط مع API واتساب الخارجي بنظام جدولة متقدم:

*   **حفظ الرسائل المجدولة (`whatsapp_queue`)**: لا يتم إرسال الرسائل فورياً لتجنب توقف الموقع، بل يتم تخزينها في جدول بقاعدة البيانات كطابور انتظار.
*   **التأخير العشوائي الآمن (`min_delay` & `max_delay`)**: عند الإرسال المجدول، يقوم النظام بوضع فاصل زمني عشوائي بين الرسائل (مثلاً من 3 إلى 15 ثانية) لكي تظهر الرسائل كأنها مرسلة من شخص طبيعي وتجنب حظر الرقم من قبل شركة واتساب.
*   **التشغيل التلقائي عبر الـ Cron Job**: يقوم ملف `whatsapp/cron.php` بالعمل خلف الكواليس كل دقيقة للتحقق من وجود رسائل مستحقة الإرسال وإرسالها عبر بوابة `whatsapp.enjaz.cloud`.

## نصائح هامة لتصميم نظامك القادم بنفس الطريقة:

1.  **ابدأ بإنشاء نظام الأمان والمصادقة أولاً**: ضع حجر الأساس لجلسات آمنة وحماية ضد CSRF و XSS قبل البدء بكتابة منطق العمل (Business Logic).
2.  **اجعل الكود نموذجياً (Modular)**: افصل إعدادات التطبيق عن الدوال المساعدة، واجعل لكل جدول في قاعدة البيانات مجلداً خاصاً به يحتوي على عمليات CRUD الخاصة به.
3.  **اعتمد على PDO دائماً**: لا تستخدم `mysqli` بالطريقة القديمة، فالـ PDO يوفر مرونة وأمان أكبر وإمكانيات ربط مع قواعد بيانات متعددة في المستقبل.
4.  **اهتم بالتفاصيل الدقيقة للواجهة (Micro-interactions)**: تغيير نصوص الأزرار أثناء التحميل، التنسيق المناسب للتواريخ والأرقام، وتنبيهات الأمان الآلية تضيف احترافية عالية جداً لعملك.
